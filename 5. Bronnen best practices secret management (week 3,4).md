Vanuit het github onderzoek is gebleken dat er slecht wordt omgegaan met credentials. Alhoewel het niet terug te zien is in de geanalyseerde code, is security by design een algemeen geacepteerd principe. iedereen die code ontwikkeld, al helemaal voor een bedrijf zou hiervan op de hoogte moeten zijn.

Toch moet dit onderbouwd worden met literatuur. API-keys vallen onder credentials, een andere benaming hiervoor is een secret. We gaan opzoek naar literatuur die best practices definieert voor secret management. We maken onderscheid tussen onafhankelijke instantie OWASP en leveranciers

De volgende bronnen worden hiervoor aangehaald:
Best practices for managing AWS Secrets Manager secrets.  [https://docs.aws.amazon.com/secretsmanager/latest/userguide/intro.html](https://docs.aws.amazon.com/secretsmanager/latest/userguide/intro.html)
AWS secret manager is een breed geïmplementeerde en geaccepteerde secret manager. De guide bevat logscherwijs relevante informatie over de implementatie van vaults/secret managers. 

Vault architecture and security model.  [https://developer.hashicorp.com/vault/docs/internals/architecture](https://developer.hashicorp.com/vault/docs/internals/architecture)
Hashicorps Vult is een breed geïmplementeerde en geaccepteerde secret manager. De guide bevat logscherwijs relevante informatie over de implementatie van vaults/secret managers. 

OWASP Top 10 for Large Language Model Applications_. [https://owasp.org/www-project-top-10-for-large-language-model-applications/](https://owasp.org/www-project-top-10-for-large-language-model-applications/)
AI-agents zijn gewoon LLM's. we halen de OWASP top 10 voor LLM aan om de relevantie van het MCP risico. het gebruik van MCP valt onder de supply chain en foutieve code die hierdoor het bedrijf in komt valt onder een supply chain attack. 

Secrets Management Cheat Sheet[https://cheatsheetseries.owasp.org/cheatsheets/Secrets_Management_Cheat_Sheet.html](https://cheatsheetseries.owasp.org/cheatsheets/Secrets_Management_Cheat_Sheet.html)
Heel handig document met veel algemene en hele relevante informatie op het gebied van secret managament.

De bovenstaande bronnen vormen de basis voor het in kaart brengen van de best prectices voor secret management.
