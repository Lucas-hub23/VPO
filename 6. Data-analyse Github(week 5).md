De eerste stap voor de analyse is de open codering. aan de verzamelde data uit de repo worden codes geplakt, repo, beschrijving, code snippet, dork en type bestand.

De data over de github repos is in excel geplaatst. er zijn drie categorien te onderscheiden binnen de verzamelde code snippets. Dit onderscheid is gemaakt door de verzamelde code verder te analyseren. De volgende fenomenen vielen op tijdens het analyseren: er waren keys direct in code te vinden, aanwezigheid van templates die uitnodigen tot hardcoding en documentatie die aanspoort tot hardcoding.

De data is na de open codering axiaal gecodeerd. Alle gevonden repo's hebben een van deze categorie toegewezen gekregen: direct data exposure, insecure templating en insecure documenting. Deze data is visueel verwerkt in de vorm van een excel.
