Uit de literatuur zijn er 3 thema's onderscheiden om aan de kwetsbaarheden gevonden in de broncode analyse te koppelen.

* supply chain attack
Aanvallen door software geleverd vanuit software gebruikt door derden
* Hardcoding
Daadwerkelijke api-keys in statische configuratie bestanden of broncode
* statische configuratiebestanden met api keys
Configuratiebestanden waarbij credentials in plain tekst worden opgeslagen in het configuratie bestand, een no go in de cyber wereld.

Vanuit deze thema's is er een nieuwe analyse uitgevoerd. Binnen deze analyse is de broncode opnieuw geanalyseerd. Alle repo's en de bijbehorende categorien zijn gekoppeld aan een thema.

Uit de literatuur zijn ook twee best practice thema's onderscheiden

* vault/credential manager
Het gebruiken van een centrale kluis voor authenticatie en uitgeven van credentials
* emphareal tokens
Een credential maar in tijdelijke vorm wordt uitgegeven door een vault, maar is maar voor een beperkte tijd bruikbaar.
